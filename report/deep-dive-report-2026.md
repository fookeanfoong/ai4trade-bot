## 1. 这份报告怎么用

「中转站」(AI API aggregator / proxy provider)通过在海外部署节点、代理转发到 OpenAI / Anthropic / Google 等上游,同时提供本地付款(支付宝/微信/USDT)与折扣价。它们能省钱、能解决国内访问与支付,但也是重灾区:**模型偷偷降级、账单对不上、预充值余额随跑路蒸发**。

本报告的目的不是替你选一家,而是给你一套**可复现的判断方法** + **逐家核对过的事实**,让你自己在任何时点都能选对。

## 8. 5 个踩雷失败模式(附识别方法)

*这些是行业内**通用的失败模式**,不针对任何具体公司;学会识别,任何一家都能自保。*

### 案例 1:预充值余额蒸发(跑路)
- **怎么发生**:高折扣吸引大额充值 → 上游被封或老板跑路 → 余额一夜清零,无人对接。
- **识别信号**:异常低价、要求大额起充、无发票、运营时间短、客服只在群里。
- **自保**:小额多次充,余额别囤;优先选有发票、运营久的。

### 案例 2:模型偷偷降级
- **怎么发生**:你调 Opus/GPT-4o,后台悄悄路由到便宜/量化小模型,你付高价拿缩水货。
- **识别信号**:同一 prompt 质量忽高忽低;价格低到不合理。
- **自保**:定期用固定基准 prompt 抽查输出质量;比对官方直连结果。

### 案例 3:计费倍率暗加价
- **怎么发生**:面板对某些模型设隐藏倍率,标价一样、实际扣费更多。
- **识别信号**:充 100 用起来「不经花」,账单 token 数与你估算对不上。
- **自保**:小额充值后,用已知 token 量的请求核对扣费。

### 案例 4:信用卡盗刷 / 拒付连累
- **怎么发生**:平台混入盗刷卡交易,支付通道风控波及正常用户,账号/余额被冻。
- **识别信号**:支付方式杂乱、风控松、社区有冻结投诉。
- **自保**:优先正规支付(支付宝/官方渠道),留好充值凭证。

### 案例 5:限流与「无限量」文字游戏
- **怎么发生**:宣传高并发/不限量,实际到量即降速或报错,退款无门。
- **识别信号**:条款里「合理使用」定义模糊;高峰频繁 429。
- **自保**:上线前压测你的真实峰值;把限流条款截图存档。
